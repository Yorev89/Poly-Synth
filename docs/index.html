The Project

What I built — and where it stands

📷  Photo placeholder — main synth hardware — August 2025

The project started as an analog design using the AD9833 waveform generator, then pivoted to a fully digital architecture when the limitations of single-channel synthesis became clear. The current design runs on three microcontrollers: an STM32F411 as the synthesis engine driving a PCM5102A DAC over I2S, and two ESP32 modules handling control, connectivity, and Bluetooth MIDI.

Tools & Resources

What actually helped

Not a comprehensive list. These are the tools and references I personally used — with straightforward notes on where they help and where they don't.

Design & development tools
EasyEDA
Used for all schematics and PCB layout. Browser-based, no installation required, handles the full workflow from schematic to Gerber export. Accessible without a professional EDA background.
Arduino IDE
Used for all firmware across all three MCUs — ESP32 and STM32 targets with the appropriate board packages. One IDE for the whole project kept things consistent.
Claude (Anthropic)
Used for the majority of code generation, debugging, and working through architectural decisions. AI assistance was central to the project — a way to move through problems that would otherwise have taken much longer. The quality of what you get back depends heavily on how clearly you describe what you're trying to do and what's going wrong.
Reference documentation
STM32F411 Reference Manual
Dense, but the authoritative source on the chip's peripherals. Search for the specific section you need rather than reading linearly.
Espressif ESP32 Technical Reference
Better organized than expected. The I2S, BLE, and memory management sections were all useful at different points in this project.
PCM5102A Datasheet
Essential for understanding I2S configuration and the DAC's filter modes. Read alongside an I2S tutorial if you're new to the protocol.
Communities
r/synthdiy
Haven't used it personally during this project, but it's active and covers a wide range of skill levels. Worth knowing it exists.
Elektronauts forum
More advanced, synthesis-focused. Useful for deeper questions about voice architecture and sound design.

What's Next

The current frontier

In progress

Automatic music generation

The next phase is making the synth generate its own music — algorithmically, driven by external inputs such as sensors. The three-MCU architecture maps cleanly onto this without modification: the STM32 continues as the pure synthesis engine, the first ESP32 generates local MIDI sequences using a Markov chain melody generator, and the second ESP32 acts as a gateway to AI-generated MIDI from the cloud.

The three-MCU split, originally forced by a hardware constraint, turned out to be exactly the right architecture for what comes next.

Under consideration

Instrument preset emulation

A longer-term question: can the synth emulate real musical instruments convincingly? Realistic instrument emulation — through physical modeling or detailed wavetables — is computationally heavier than the current subtractive synthesis engine. The STM32F411 may not have enough headroom for this without an upgrade to a more powerful MCU. It's an open question worth exploring, not a committed roadmap item.

This site updates as the project evolves. The build log is written as things happen — including the parts that don't work yet.
